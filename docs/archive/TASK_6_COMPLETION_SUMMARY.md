## 💡 Best Parts

### 1. Integration-First Philosophy
```
Real dependencies > mocks
Real temp dirs > mocked fs
Real Next.js builds > mocked compilation
Real browsers > JSDOM snapshots
```

### 2. 5-Layer Testing Pyramid
```
5. Schema Fuzzing (chaos engineering)
4. Performance Benchmarks (budgets enforced)
3. Accessibility Testing (WCAG 2.1 AA)
2. Visual Regression (screenshot comparison)
1. Unit + Integration Tests (behavior verification)
```

### 3. Strategic Coverage Targets
Not blind 100%, but smart:
- 90% on `@stackwright/types` (schemas are the contract)
- 80% on `@stackwright/core` (critical rendering path)
- 75% on `@stackwright/build-scripts` (complex logic)
- 60% on `@stackwright/themes` (mostly CSS passthrough)

### 4. Performance Budgets with Rationale
Every budget has a "why":
- FCP < 1.5s → Fast perceived load
- LCP < 2.5s → Content visible quickly
- CLS < 0.1 → Minimal layout shift
- First Load JS < 100KB → Small initial bundle

### 5. Comprehensive Troubleshooting
Every failure scenario covered:
- "Cannot find module" → rebuild all packages
- "Timeout" → increase timeout for slow tests
- "Screenshot failed" → review diff, update baseline
- "WebKit only fails" → Safari-specific CSS issue
