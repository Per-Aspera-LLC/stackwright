# @stackwright/types — Agent Guide

The grammar of Stackwright. This package is the **single source of truth** for all content types, site configuration, and theme structure. Zod schemas define the grammar; TypeScript types are inferred via `z.infer<>`.

---

## How It Works

1. **Zod schemas** in `src/types/` define every content type, field, and enum.
2. **TypeScript types** are inferred from Zod — never hand-written separately.
3. **JSON schemas** are generated from Zod via `zod-to-json-schema` for IDE YAML validation.
4. **Runtime validation** uses the Zod schemas directly in the prebuild pipeline.
5. **MCP tools** introspect schemas at runtime via `schema.def` (Zod v4).

---

## Source Files

```
src/types/
  base.ts               — BaseContent (label, color, background)
  content.ts            — All content type schemas (main, carousel, faq, etc.)
  collection.ts         — CollectionConfig schema (_collection.yaml)
  enums.ts              — TypographyVariant, ButtonVariant, etc.
  layout.ts             — PageContent, pageContentSchema
  media.ts              — MediaItem discriminated union (media | icon | image)
  media-primitives.ts   — ImageContent, IconContent base schemas
  navigation.ts         — Navigation, menu item schemas
  siteConfig.ts         — SiteConfig schema (theme, nav, app bar, footer, SEO)
  index.ts              — Re-exports everything
  yaml.d.ts             — js-yaml type declarations
```

---

## JSON Schema Generation

Schemas are generated by `src/generate-schemas.ts` using Zod v4's built-in `z.toJSONSchema()`:

```bash
cd packages/types && pnpm generate-schemas
```

This produces three files in `dist/schemas/`:
- `theme-schema.json` — Theme configuration validation
- `content-schema.json` — Page content structure validation
- `siteconfig-schema.json` — Site-wide configuration validation

These are published as package exports for IDE YAML validation.

---

## Content Type Maintenance Rule

**When modifying anything in `src/types/`** — adding, removing, or changing any content type, field, or enum — you MUST:

1. Run `pnpm stackwright -- generate-agent-docs` to regenerate the auto-generated tables in all `AGENTS.md` files
2. Run `cd packages/types && pnpm generate-schemas` to regenerate JSON schemas
3. Update or add unit tests in `packages/core/test/`
4. Verify E2E tests pass (`pnpm test:e2e`)

**Do NOT edit the content between `<!-- stackwright:content-type-table:start/end -->` markers manually.** Those tables are auto-generated from the live Zod schemas.

## Dependencies

- **js-yaml** — YAML parsing for tests
- **fs-extra** — File operations for schema generation script

## Build

tsup produces ESM (`.mjs`) + CJS (`.js`) + type declarations. Schema JSON files are copied to `dist/schemas/` during build.

```bash
pnpm build:types     # Build from monorepo root
pnpm generate-schemas  # Regenerate schemas only (from this package)
```
